# Organic tree — visual scenarios

## Desktop

1. `nearby` — выбранный человек в центре, родители ниже, дети выше.
2. `generations` — три уровня читаются без пересечений.
3. `branch` — линия рода подсвечивается, боковые ветви приглушены.
4. `all` — несколько семейных компонентов не слипаются.
5. Переключение `Дерево ↔ Схема` сохраняет focus и selection.
6. `overview / compact / full` меняют только детали, а не позиции.
7. Длинное ФИО не вылезает за медальонную табличку.
8. Человек без фото показывает монограмму.
9. Человек с фото не меняет геометрию узла.
10. Бывшие супруги имеют светлую разорванную ветвь.
11. Усыновление показывает двойную ветвь.
12. Отчим / мачеха показываются тонкой пунктирной ветвью.
13. Опекун показывает ветвь с маркерами.
14. Неуточнённая связь показывает `?`.
15. Свернутая ветка показывает декоративный badge с числом скрытых людей.

## Mobile

1. `nearby` открывается без попытки уместить всю семью.
2. Портреты кликабельны, pan/zoom не конфликтует с тапом.
3. Bottom sheet профиля открывается как раньше.
4. Toolbar не перекрывает центральный медальон.
5. На маленьком zoom скрываются подписи, но остаются медальоны.
6. Миникарта остаётся компактной и не мешает дереву.

## Realtime / roles

1. Обновление фотографии меняет только медальон.
2. Обновление биографии не пересчитывает layout.
3. Relative / editor / admin видят те же доступные действия в профиле.
4. Search → выбор человека корректно центрирует organic tree.
