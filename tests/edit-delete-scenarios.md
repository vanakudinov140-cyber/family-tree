# Сценарии проверки edit / delete

Ручные сценарии (Playwright не обязателен).

## Admin редактирует имя

1. Войти как admin.
2. Открыть карточку → Редактировать → изменить имя → Сохранить.
3. Ожидание: тост «Данные сохранены», имя обновилось в карточке и панели.

## Relative не может редактировать

1. Войти как relative (или без входа).
2. Открыть карточку.
3. Ожидание: нет кнопок «Редактировать» / «Удалить человека».
4. Прямой вызов RPC `update_person` / `delete_person` должен вернуть ошибку прав.

## Admin удаляет тестового родственника

1. Добавить тестового человека.
2. Удалить его через диалог с подтверждением `УДАЛИТЬ`.
3. Ожидание: тост «Человек удалён», человек исчез из дерева.

## При удалении исчезают relationships

1. Перед удалением запомнить число связей человека.
2. После удаления в SQL:

```sql
select count(*) from public.relationships
where person1_id = '<deleted-id>' or person2_id = '<deleted-id>';
```

Ожидание: `0`.

## Нельзя удалить текущий центр

1. Открыть центрального человека (метка «Это вы» / currentUserId).
2. Нажать «Удалить человека».
3. Ожидание: сообщение «Сначала выберите другого центрального человека», удаление недоступно.

## Форма не отправляется дважды

1. В Edit/Delete нажать сохранить/удалить и сразу кликнуть повторно.
2. Ожидание: кнопка disabled, один запрос.

## Realtime UPDATE / DELETE

1. Две вкладки.
2. UPDATE в одной → обновление во второй.
3. DELETE в одной → исчезновение во второй; открытый профиль удалённого закрывается.
