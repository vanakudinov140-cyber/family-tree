# Сценарии проверки: связи и импорт семьи

Playwright не обязателен, если Chromium недоступен. Ниже — ручные / SQL / UI проверки.

## 1. Валидный импорт двух людей и одной связи

Подготовить минимальный JSON с двумя `people` и одной `relationships`.  
Admin → проверить → импортировать с фразой `ИМПОРТИРОВАТЬ`.  
Ожидание: 2 новых человека, 1 связь, дерево обновилось.

## 2. Повторный импорт — ноль новых записей

Повторить тот же файл.  
Ожидание: `skippedPeople` / `skippedRelationships`, без дубликатов.

## 3. Неизвестный personKey — ошибка

Связь ссылается на ключ, которого нет в payload и в базе.  
Ожидание: `valid: false`, ключ в `unresolvedKeys`, импорт недоступен.

## 4. Человек — собственный родитель — ошибка

`person1Key === person2Key` или parent на себя.  
Ожидание: ошибка валидации.

## 5. Родительский цикл — ошибка

A → B → A (через parent).  
Ожидание: ошибка цикла, импорт отклонён.

## 6. Дублирующая spouse-связь в обратном порядке

Уже есть A–B spouse, в файле B–A.  
Ожидание: предупреждение или skip при импорте (`reverse_or_existing_spouse`), без второй строки.

## 7. Relative не может импортировать

Войти как relative.  
Ожидание: пункта «Импорт семьи» нет; прямой RPC `import_family_batch` / `validate_family_import` возвращает отказ admin-only.

## 8. При ошибке нет частичного импорта

Подготовить файл, где люди валидны, но связь создаёт цикл.  
Ожидание: исключение / отказ; новые люди из этого batch не появляются.

## 9. Множественные браки корректно отображаются

Человек с двумя spouse и детьми от разных союзов.  
Ожидание: отдельный union на каждый союз; супруги не соединены друг с другом напрямую; дети под своим союзом.

## 10. Step / adoptive / biological визуально различаются

- biological — сплошная линия;
- adoptive — мягкий пунктир;
- step — более светлая точечная;
- guardian — пунктир + подпись «опека»;
- former spouse — более светлая линия;
- probable/uncertain — «?» или тонкий пунктир.

Кнопка **Типы связей** открывает легенду (на мобильном — нижняя панель).
