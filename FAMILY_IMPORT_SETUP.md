# Настройка расширенных связей и импорта семьи

## 1. Выполнить миграцию 004

1. Откройте Supabase → SQL Editor.
2. Скопируйте содержимое файла  
   `supabase/migrations/004_relationship_types_and_family_import.sql`
3. Выполните скрипт целиком.
4. Убедитесь, что нет ошибок.

Миграция добавляет поля к `people` / `relationships` и RPC:

- `validate_family_import(payload jsonb)` — только проверка;
- `import_family_batch(payload jsonb)` — атомарный импорт в режиме `insert_only`.

Не используйте service_role в приложении. SQL выполняйте вручную.

## 2. Скачать backup

Войдите как admin → меню пользователя → **Скачать резервную копию**.

Файл: `family-tree-backup-YYYY-MM-DD-HH-mm.json`  
Содержит people, relationships, дату и версию схемы.  
Не содержит Auth-пользователей, email, токены и секреты.

## 3. Заполнить `family-import-draft.json`

1. Скопируйте структуру из `family-data/family-import-template.json`.
2. Заполняйте `family-data/family-import-draft.json` по README в `family-data/`.
3. Используйте стабильные `key`.
4. Не придумывайте неизвестные даты и родственные связи.

## 4. Проверить файл без импорта

Admin → **Импорт семьи** → вставить JSON или выбрать файл → **Проверить файл**.

На этом шаге в базу ничего не записывается.

## 5. Ошибки и предупреждения

- **Ошибки** блокируют импорт (циклы, неизвестные ключи, самосвязь и т.п.).
- **Предупреждения** не блокируют (needs_review, probable/uncertain, совпадение имён).
- Совпадение ФИО — только предупреждение; объединение только по `external_key`.

## 6. Выполнить импорт

1. Убедитесь, что проверка прошла без ошибок.
2. Введите точно: `ИМПОРТИРОВАТЬ`
3. Нажмите **Импортировать**.
4. Дождитесь отчёта. Окно не закрывается при ошибке.

## 7. Повторный импорт

Повторный запуск того же файла:

- люди/связи с уже существующим `external_key` пропускаются;
- данные не перезаписываются;
- новых дубликатов не создаётся.

## 8. Восстановление из backup

Backup — снимок для ручного восстановления.  
На этом этапе нет отдельной кнопки «восстановить».  
При необходимости используйте backup как справочник и аккуратно переносите записи через импорт/`insert_only` или ручные правки admin-инструментами.  
Перед любым массовым изменением снова скачайте свежий backup.

## 9. Почему сначала проверить имена и связи всей семьи

Ошибки в `key` и parent/spouse связях размножаются на всё дерево.  
Сначала сверка на бумаге/в черновике, потом «Проверить файл», и только затем импорт.

## 10. Почему неизвестные даты должны оставаться пустыми

Выдуманные даты выглядят как факты и портят историю семьи.  
Оставляйте `null` и при необходимости ставьте `needs_review` / `uncertain`.
