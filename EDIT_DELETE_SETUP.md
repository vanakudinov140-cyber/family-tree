# Редактирование и удаление родственников

Инструкция после добавления миграции `003_edit_and_delete_people.sql`.

Секретные ключи здесь не приводятся.

---

## 1. Выполнить миграцию

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard) → ваш проект.
2. Перейдите в **SQL Editor**.
3. Откройте файл:

   `supabase/migrations/003_edit_and_delete_people.sql`

4. Скопируйте весь SQL и выполните (**Run**).

Миграция создаёт RPC:

- `public.update_person`
- `public.delete_person`

Обе функции доступны только авторизованному администратору (`public.is_admin()`).

Повторный запуск безопасен.

---

## 2. Проверить редактирование

1. Войдите как admin.
2. Откройте карточку любого человека.
3. Нажмите **Редактировать**.
4. Измените, например, отчество или биографию.
5. Сохраните.
6. Должно появиться уведомление «Данные сохранены».
7. Карточка и панель профиля обновятся.

Если вы relative или не вошли — кнопок редактирования/удаления нет, а RPC вернёт ошибку прав.

---

## 3. Проверить удаление тестового родственника

1. Сначала добавьте тестового родственника (например, брата/сестру к дальнему родственнику).
2. Откройте его карточку.
3. Нажмите **Удалить человека**.
4. Введите слово `УДАЛИТЬ` точно в таком виде.
5. Подтвердите удаление.
6. Должно появиться «Человек удалён».
7. Панель профиля закроется, дерево перестроится.

Нельзя удалить:

- единственного человека в дереве;
- текущего центрального человека, пока центр не сменён (сообщение: «Сначала выберите другого центрального человека»).

---

## 4. Проверить вторую вкладку (Realtime)

1. Откройте сайт в двух вкладках.
2. В одной вкладке отредактируйте человека.
3. Во второй карточка должна обновиться без перезагрузки.
4. Удалите тестового человека в первой вкладке.
5. Во второй он исчезнет; если его профиль был открыт — панель закроется.

---

## 5. Почему удаление нельзя отменить

`delete_person` делает **физическое** удаление:

- строки в `relationships`, где человек участвует;
- затем строки в `people`.

Это не «архивация» и не soft-delete. Без резервной копии вернуть данные нельзя.

---

## 6. Резервная копия перед удалением

В SQL Editor можно сохранить снимок:

```sql
-- Создать резервные таблицы (однократно перед опасными действиями)
create table if not exists public.people_backup_manual as
select * from public.people where false;

create table if not exists public.relationships_backup_manual as
select * from public.relationships where false;

truncate public.people_backup_manual;
truncate public.relationships_backup_manual;

insert into public.people_backup_manual
select * from public.people;

insert into public.relationships_backup_manual
select * from public.relationships;
```

Либо экспортируйте таблицы через Dashboard → Table Editor → Export.

---

## 7. Если RPC говорит, что пользователь не администратор

1. Проверьте роль:

```sql
select id, email, role from public.profiles order by updated_at desc;
```

2. При необходимости выполните снова `supabase/promote_first_admin.sql` с вашим email.
3. Выйдите из сайта и войдите снова.
4. В меню пользователя роль должна быть «администратор».

Также убедитесь, что миграции `002` и `003` выполнены.
