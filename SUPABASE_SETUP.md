# Подключение Supabase к Family Tree

Этот этап только читает родственников и связи из базы.  
Авторизация, редактирование и загрузка фотографий пока не используются.

## 1. Создайте проект Supabase

1. Откройте [https://supabase.com](https://supabase.com) и войдите в аккаунт.
2. Нажмите **New project**.
3. Укажите имя проекта, пароль базы данных и регион.
4. Дождитесь создания проекта.

## 2. Выполните SQL-миграцию

1. В панели Supabase откройте **SQL Editor**.
2. Создайте новый запрос.
3. Скопируйте содержимое файла:

`supabase/migrations/001_initial_family_schema.sql`

4. Нажмите **Run**.

Скрипт создаст таблицы `people` и `relationships`, индексы и политики RLS только для чтения.

## 3. Загрузите тестовые данные

1. Снова откройте **SQL Editor**.
2. Скопируйте содержимое файла:

`supabase/seed.sql`

3. Нажмите **Run**.

В базе появятся 6 человек семьи Третьяковых и их связи.

| Иван Михайлович | `33333333-3333-4333-8333-333333333333` |
| Дина Сергеевна | `44444444-4444-4444-8444-444444444444` |

Тестовый персонаж «Демид Иванович Третьяков» удалён из seed. Если он остался в старой базе, выполните `supabase/scripts/remove-test-demid-tretyakov.sql`.

## 4. Найдите URL проекта и anon key

1. Откройте **Project Settings → API**.
2. Скопируйте:
   - **Project URL** → это `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public** key → это `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### Важно про ключи

- В клиентское приложение добавляется **только anon key**.
- **service_role key нельзя** добавлять в клиентское приложение.
- **service_role key нельзя** хранить в переменных `NEXT_PUBLIC_*`.
- service_role обходит RLS и подходит только для защищённого серверного кода.

## 5. Создайте `.env.local`

В корне проекта `family-tree` создайте файл `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

Можно скопировать шаблон:

```bash
cp .env.local.example .env.local
```

Затем вставьте свои значения.

## 6. Запустите приложение

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## 7. Как понять, откуда берутся данные

Приложение само выбирает источник:

1. Если `NEXT_PUBLIC_SUPABASE_URL` или `NEXT_PUBLIC_SUPABASE_ANON_KEY` пустые — используются локальные данные из `src/data/family.ts`.
2. Если переменные заданы — приложение запрашивает `people` и `relationships` из Supabase.
3. Если запрос к базе падает или таблица пустая — снова включается локальный резервный режим.

В режиме разработки при локальных данных показывается уведомление:

**«Используются локальные тестовые данные»**

Если данные пришли из Supabase, дерево выглядит так же, как локальная версия, но ID людей будут UUID из seed.

## 8. Быстрая проверка в Supabase

В SQL Editor можно выполнить:

```sql
select count(*) from public.people;
select count(*) from public.relationships;
```

Ожидается:

- `people` → 7
- `relationships` → 10
